Core Errors

0 NON_ERROR STATUS
B, B, B, B
Message(none)
DescriptionNo error. The device is operating normally.
ActionNo action required. This is the default state.
-1 WIFI_AP_CONNECT_FAIL ERROR
R, R, R, R
MessageConnect fail.
DescriptionThe device failed to connect to the configured Wi-Fi access point. This may be caused by incorrect credentials, weak signal, or the AP being unavailable.
How to Fix
  1. Verify the Wi-Fi SSID and password are correct.
  2. Ensure the Wi-Fi access point is powered on and within range.
  3. Restart the device and try again.
  4. Re-run device setup if the issue persists.
-2 DOWNLOAD_PLAN_FAIL ERROR
R, R, R, B
MessageGet schedule from CMS error.
DescriptionThe device failed to download the schedule/plan from the CMS server. The server may be unreachable or returned an error.
How to Fix
  1. Check the device's network connection.
  2. Verify the CMS server is online and accessible.
  3. Check CMS settings and ensure a valid plan is assigned to this device.
  4. Contact your CMS administrator if the problem persists.
-3 DOWNLOAD_PICTURE_FAIL ERROR
R, R, B, R
MessageDownload Fail
DescriptionThe device failed to download one or more images from the CMS server.
How to Fix
  1. Check the device's network connection.
  2. Verify the image files exist on the CMS server.
  3. Check available storage on the device.
  4. Retry by restarting the device.
-4 NO_PLAN_IN_CMS ERROR
R, R, B, B
MessageNo plan in CMS
DescriptionThe CMS server responded successfully but returned no plan/schedule data for this device.
How to Fix
  1. Log in to the CMS and assign a schedule to this device.
  2. Ensure the device is registered correctly in CMS.
  3. Verify the schedule time range covers the current date/time.
-5 MAC_ADDRESS_NOT_REGISTERED ERROR
R, B, R, R
MessageDevice not register
DescriptionThe device's MAC address is not registered in the CMS database. The server rejected the device.
How to Fix
  1. Log in to the CMS and add this device's MAC address.
  2. Verify the MAC address matches the one shown on the device.
  3. Restart the device after registration.
-6 PICTURE_CHECKSUM_FAIL ERROR
R, B, R, B
MessagePicture checksum failed
DescriptionA downloaded image file's checksum does not match the expected value. The file may be corrupted or partially downloaded.
How to Fix
  1. Check the network connection stability.
  2. Re-upload the image on the CMS server.
  3. Restart the device to trigger a re-download.
-7 LOW_BATTERY STATUS
R, B, B, B
MessageLow Battery
DescriptionThe device battery level is critically low. The device may shut down soon.
Action
  1. Connect the device to a power source immediately.
  2. Allow the device to charge before resuming operation.

Network & NAS

443 CONNECT_FAIL_WITH_NO_GET_MAC_ADDRESS ERROR
R, G, R, G
MessageConnect fail. No get MAC-address. Re-connecting...
DescriptionWi-Fi connection failed and the device could not retrieve its own MAC address. The device is attempting to reconnect automatically.
How to Fix
  1. Verify Wi-Fi AP is available and within range.
  2. Restart the device.
  3. If the issue persists, re-run the initial device setup.
444 RESPONSE_PARSING_ERROR ERROR
R, R, G, B
MessageResponse message is parsing error
DescriptionThe CMS server response could not be parsed. The response format may be invalid or corrupted during transmission.
How to Fix
  1. Check the network connection for stability.
  2. Verify the CMS server is running the correct version.
  3. Contact your CMS administrator to check server logs.
-8 DOWNLOADING STATUS
B, B, B, B

DescriptionThe device is currently downloading content. This is a normal status indicator, not an error.

  4. Restart the device and try again.

-11 NAS_ZIP_ERROR ERROR
R, B, R, B
MessageNAS zip file error
DescriptionThe ZIP file downloaded from the NAS is corrupted or cannot be extracted.

-12 OTA_DOWNLOADING STATUS
B, B, B, B
MessageUpdate via OTA

-13 NO_AVAILABLE_SCHEDULE ERROR
R, G, G, R
MessageNo Available Schedule
DescriptionThe device has no playable schedule for the current date and time. A schedule may have expired or none has been assigned.
How to Fix
  1. Log in to the CMS and verify a schedule is assigned to this device.
  2. Check that the schedule covers the current date and time range.
  3. Assign a new schedule if the previous one has expired.
-14 IMAGE_WRONG_SIZE ERROR
R, G, G, B
MessageImage is wrong size.
DescriptionA downloaded image has incorrect dimensions that do not match the device's display resolution.

  2. Re-upload correctly sized images to the CMS.
  3. Restart the device after fixing the images.

-41OTA_INFO_REQUEST_FAILOTA
G, G, R, G
MessageOTA info request failed.
DescriptionThe device received an OTA update job from CMS, but the OTA server URL was empty or the update information JSON could not be parsed. The OTA process was aborted at the info-gathering stage before any download was attempted.
How to Fix
  1. Check the CMS OTA job configuration — ensure the OTA URL field is not empty.
  2. Verify the OTA server is online and the info JSON files (APKInfo.json, BSPInfo.json, MCUInfo.json) are accessible.
  3. Check the device log for details on which info request failed.
  4. Re-send the OTA job from CMS after fixing the configuration.
-42OTA_INFO_REQUEST_TIMEOUTOTA
G, G, G, R
MessageOTA info request timeout.
DescriptionThe device sent HTTP requests to the OTA server to retrieve update version information (APK/BSP/MCU info), but did not receive all responses within the 5-minute timeout period. This typically indicates a network connectivity issue between the device and the OTA server.
How to Fix
  1. Check the device's network connection (Wi-Fi signal strength, LTE signal, etc.).
  2. Verify the OTA server is online and responding to HTTP requests.
  3. Check if a firewall or proxy is blocking access to the OTA server URL.
  4. Restart the device and wait for CMS to re-send the OTA job.

Hardware & System

-50HARDWARE_REVISION_ERRORHARDWARE
R, G, B, R
MessageHardware Revision Error
DescriptionThe device hardware revision does not match the expected configuration. The firmware may be incompatible with this hardware.
How to Fix
  1. Verify the correct firmware version for this hardware revision.
  2. Contact the hardware vendor for compatibility information.
  3. Reinstall the appropriate firmware version.
-51RESET_TO_DEFAULTSYSTEM
R, G, B, G
MessageReset to default!
DescriptionThe device has been reset to factory default settings. All configuration has been cleared.
Action
  1. Re-configure the device with the correct settings.
  2. Re-connect to Wi-Fi and CMS.
  3. Re-assign schedules in CMS if needed.
-52APP_NOT_FOUNDSYSTEM
R, G, G, G
MessageAPP NOT FIND !
DescriptionA required application was not found on the device. The app may have been uninstalled or not properly installed.
How to Fix
  1. Reinstall the required application via OTA or manual installation.
  2. Verify the app package name is correct.
  3. Contact support for assistance.
